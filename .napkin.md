## Patterns That Don't Work
- **MERIT DEM WSE as a flow-direction signal**: Always has data so dominates multi-signal votes, but is low-quality on flat rivers. SWOT is better but sparse. When they disagree, SWOT is usually right (AS 18081).
- **OLS for node-level WSE slope**: Outlier clusters (e.g., 14 bad SWOT nodes) dominate. Use Theil-Sen instead.
- **Automated flow-direction flipping at scale**: 20 approved out of 2,032 candidates (1%). Most WSE inversions aren't from reversed edges — they're DEM artifacts, lake interactions, backwater, or tidal effects.

## Domain Notes
- **pathlen_hw semantics**: Cumulative reach_lengths from headwater→self (self included, headwater excluded). Headwater gets 0. Computed in upstream topological pass.
- **pathlen_out semantics**: Cumulative reach_lengths from self's successor→outlet (all included, self excluded). Outlet gets 0. Computed in downstream topological pass.
- **pathlen_hw + pathlen_out = constant** along the same (best_headwater, best_outlet) path. Constant = total_path_length - headwater_reach_length.
- **pathlen_hw step violations (V016)**: 1,550 globally, ALL at confluences (n_rch_up > 1), 0 on 1:1 links. Caused by upstream pass picking a different best predecessor than rch_id_dn_main implies. Expected routing divergence.
- **pathlen_out step violations (V017)**: 0 globally. pathlen_out and rch_id_dn_main both computed in downstream direction, so they always agree.
- **V019 direction violations**: 737 total — 661 involve side channels (main_side > 0), 76 main-to-main. Side-channel violations are expected routing divergence at rejoins.

- **N013 (CL-node misallocation) is 99.7% resolved**: v17b had 89,364 violations, v17c has 311. Fixed by `sync_centerline_node_ids()` (commit 7b0ca71) using cl_id_min/max range reassignment. Remaining 311 are structural (node sparsity). Spatial-nearest fix evaluated but rejected — breaks cl_id_min/max contiguity on 26/59 reaches.
- **Pfafstetter continent codes for globally unique IDs**: AF=1, EU=2, AS=3, OC=5, SA=6, NA=7 (from first digit of reach_id). Shared constant in `src/sword_v17c_pipeline/pfaf_offsets.py`. Use `pfaf_offset(region)` to get offset (e.g. NA→7,000,000). Applied to main_path_id, path_segs, subnetwork_id. `network` (v17b from UNC) stays untouched.
- **subnetwork_id vs network**: Different variables. Both are weakly connected components but with different enumeration order (only 0-2% match). `network` is UNC's v17b original. `subnetwork_id` is our v17c computation — now globally unique with Pfafstetter offsets.
- **SWORD reused node IDs across versions** — same node_id can map to completely different rivers/coordinates between v17 releases. Always validate node coordinates spatially when joining across data sources. African rivers (ANJOBONY, BEMARIVO, ELE, V7_2, LILONGWE, SARARE) are affected; SA rivers are fine because their geometry didn't change.
- **Distance column semantics (post-split)**: `dist_out` = v17b original, `dist_out_dijkstra` = Dijkstra shortest to ANY outlet, `hydro_dist_out` = mainstem walk via rch_id_dn_main (includes own reach_length). `hydro_dist_hw` dropped (= pathlen_hw + reach_length). Mississippi spot-check: hydro=3,265km, dijkstra=3,039km (226km diff from mainstem routing). V001=0 violations, V002=0 violations globally (248,673 reaches).
- **Routing consistency rule**: All functions that pick downstream/upstream neighbors must use the same 3-tuple ranking: `(effective_width, log_facc, pathlen + reach_length)`. Functions: `compute_best_headwater_outlet`, `compute_main_neighbors`. If they diverge, `hydro_dist_out ≠ pathlen_out + reach_length` at width/facc-tied bifurcations.
